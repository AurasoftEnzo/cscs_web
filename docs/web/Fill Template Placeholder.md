# Fill Template Placeholders

## Purpose
CSCS_WPF supports two ways of replacing placeholder values in a loaded HTML template:
- `FillTemplateFromDictionary(templateHndl, valuesDictionary)`
- `FillTemplatePlaceholder(templateHndl, placeholderName, newValue)` 

Use these functions after `LoadTemplate(...)` and before `RenderHtml(...)`.

## Functions covered
| Function | Purpose |
|---|---|
| `FillTemplateFromDictionary(...)` | Replaces multiple placeholders using dictionary values |
| `FillTemplatePlaceholder(...)` | Replaces one specific placeholder |

## FillTemplateFromDictionary

### Purpose
Replaces placeholders in the template using key-value pairs from a dictionary.  
Dictionary keys must match placeholder names in the HTML template.

### Syntax
```internal-script
FillTemplateFromDictionary(templateHndl, valuesDictionary);
```

### Arguments
| Argument | Type | Required | Description |
|---|---|---|---|
| `templateHndl` | integer | yes | Handle of previously loaded template |
| `valuesDictionary` | dictionary | yes | Dictionary where keys match placeholder names |

### Returns
Returns `Variable.EmptyInstance`.

### Example
```internal-script
templateHndl = LoadTemplate("C:\\templates\\invoice.html");

values = {};
values["placeholder1"] = "value1";
values["placeholder2"] = "value2";

FillTemplateFromDictionary(templateHndl, values);
```

### Example HTML
```html
<html>
<body>
  value1 = {{placeholder1}}
  value2 = {{placeholder2}}
</body>
</html>
```

## FillTemplatePlaceholder

### Purpose
Replaces one placeholder in the template with one new value.

### Syntax
```internal-script
FillTemplatePlaceholder(templateHndl, placeholderName, newValue);
```

### Arguments
| Argument | Type | Required | Description |
|---|---|---|---|
| `templateHndl` | integer | yes | Handle of previously loaded template |
| `placeholderName` | string | yes | Name of placeholder to replace |
| `newValue` | string | yes | New string value |

### Returns
Returns `Variable.EmptyInstance`.

### Example
```internal-script
templateHndl = LoadTemplate("C:\\templates\\simple.html");
FillTemplatePlaceholder(templateHndl, "placeholder1", "value1");
```

### Example HTML
```html
<html>
<body>
  {{placeholder1}}
</body>
</html>
```

## Choosing the right function
Use `FillTemplateFromDictionary(...)` when:
- many placeholders must be replaced
- data already exists in a dictionary
- page rendering is data-driven 

Use `FillTemplatePlaceholder(...)` when:
- only one value needs to be changed
- one-off replacement is enough
- a later override of one value is needed

## Combined example
```internal-script
function renderInvoice
{
    templateHndl = LoadTemplate("C:\\templates\\invoice.html");

    values = {};
    values["invoiceNumber"] = "INV-2026-001";
    values["customerName"] = "ABC d.o.o.";
    values["totalAmount"] = "150.75";

    FillTemplateFromDictionary(templateHndl, values);
    FillTemplatePlaceholder(templateHndl, "footerText", "Generated by CSCS_WPF");

    htmlString = RenderHtml(templateHndl);

    headers = {};
    headers["Content-Type"] = "text/html";

    return Response(headers, htmlString, 200);
}
```

## Common mistakes
- Calling fill functions before `LoadTemplate(...)`.
- Using placeholder names that do not exist in the template.
- Passing non-dictionary as second argument to `FillTemplateFromDictionary(...)`.
- Expecting `FillTemplatePlaceholder(...)` to replace multiple placeholders at once.

## Best practices
- Use consistent placeholder naming in templates.
- Group page data into one dictionary where possible.
- Keep isolated overrides separate with `FillTemplatePlaceholder(...)`.
- Render only after all replacements are completed.

## AI notes
- `templateHndl` must come from `LoadTemplate(...)`.
- Placeholder names must match template markup exactly.
- Prefer dictionary-based replacement for larger pages.
- Do not confuse placeholder replacement with conditional rendering; that is handled by `RenderCondition(...)`.
